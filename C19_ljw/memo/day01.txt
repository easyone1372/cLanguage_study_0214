프로그램
개발자의 요구에 의해 개발자의 명령을 모아놓은 모음.

프로그래밍
사람: 명령 -> <-실행:컴퓨터
:사람이 명령을 내리고 컴퓨터가 실행하는 모든 과정.

C언어의 특징
1. 절차지향의 언어이다.
절차지향이란 순서에 따라 실행된다는 의미.
C언어로 작성한 프로그램은 순서를 중심으로 실행된다. 순서란 코드의 진행 순서를 이야기한다.
코드의 흐름은 위에서 아래로, 좌에서 우로 진행이 된다. 

2. 이식성이 좋다.
C언어로 작성된 프로그램은 명령을 수행하는 기계(컴퓨터)의 CPU에 구애받지 않고 실행이 가능하다.
윈도우, 맥, 리눅스 등 운영체제가 달려져도 C언어 프로그램은 큰 영향을 받지 않는다. 프로그램 언어에 따라 운영체제의 변화에 일일히 대응해야하는 경우가 있는데 C언어는 그럴 필요가 없다.

3.c언어는 저급 언어이다.
저급언어란 수준이 낮다는 의미가 아니라 컴퓨터에 조금 더 친화적인 언어라는 의미. 사용자에 친화적인 언어일수록 고급언어에 가깝다고 할 수 있다.

사람            번역가            컴퓨터
C언어 <->    컴파일러  <->  기계어

기계어: 0과 1로 이루어진 이진수 언어
0: 전류가 통하지 않는 상태
1: 전류가 통하는 상태

1. 코드 작성 -> 2. 컴파일 -> 3.링크 ->4. 디버깅 -> 5.실행
      ~~.c   -> ~~.obj   -> ~~ -> ~~ -> .exe

[변수]
변수: 값을 저장하는 공간

변수 생성방법
1. 변수의 선언과 대입
자료형 변수명; -> 변수의 선언
변수명= 데이터; -> 변수의 대입

2. 변수의 초기화(초기값을 지정한다. 초기+화)
자료형 변수명 = 데이터;

변수를 사용해야하는 이유
1. 간편한 수정
변수를 통해 여러개의 값을 한번에 수정할 수 있다.

2. 이전 명령을 기억하기 위해서

x = 2 +3;

x+4; -> 9

3. 값에 의미부여가 가능하다.

name = "홍길동"
age = 30;

변수명 명명 규칙
1. 맨앞에 숫자가 올 수 없다.
2. _(언더바)를 제외한 특수문자를 사용할 수 없다.
3.공백을 사용할 수 없다.
home address -> x
homeaddress -> 보기 불편함
스네이크표기법: home_address
카멜표기법: homeAddress

4. 이미 사용중인 키워드는 사용할 수 없다.
(printf, int, char, float, double, if, switch, while, for ...)

5. 절대 한글로 작성하면 안된다.
